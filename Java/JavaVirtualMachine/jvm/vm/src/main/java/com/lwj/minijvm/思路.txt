包含main方法的类是整个程序的入口，java XXX.class 执行某个类的时候，会先启动一个hotspot虚拟机，
虚拟机的启动会将启动类进行加载，同时会创建一个main线程来执行对应的main方法，main线程中包含了Java虚拟机栈、
PC寄存器、以及类加载器。
具体执行main方法的过程是，将main方法封装成一个栈帧，栈帧中除了包含方法具体的字节码指令，还有局部变量表、操作数栈、以及当前栈帧指令执行的位置。
main线程会一直通过程序计数器来获取下一条指令来执行，指令执行过程中，如果调用到了别的函数，同样会将对应函数封装成栈帧
压入当前线程的Java虚拟机栈顶，这个时候程序计数器获取的就是被压入方法的指令了。然后等到方法执行结束，将该方法对应的栈帧从
Java虚拟机栈顶移除，继续执行之前的栈帧指令。
具体指令执行逻辑参考JVM字节码指令规范，通过操作数栈、局部变量表、跳转命令以及返回值完成一条条字节码指令的执行。
